# ADAMS Lite

## 概要
我が家では財政原則に基づいた家計運用を行っています。官庁会計が定める予算の6原則（総計予算主義・単一予算主義・予算事前議決・会計年度独立・予算公開・予算統一）を設計の前提とし、予算の編成から決算までを体系的に管理します。

予算執行、決算といった官庁会計システムの対象業務に加えて、旅行計画、旅費精算といった旅費システムが対象とする業務機能、さらに外出先からの現金支払い即時登録（Discord連携）を導入し、会計業務を一元的に行えるシステムを構築することを目指しています。

## システム構成
ADAMS Liteは、以下のモジュールで構成されています。
- 予算管理モジュール: 予算の設定、管理、実績の追跡を行います。
- 決算管理モジュール: 決算処理、報告書の生成を行います。
- 行事計画モジュール: 行事の登録・管理・予算配分・支払計画・実績紐付けを行います。
- 基金モジュール: 基金の積立・取崩・取引履歴の管理を行います。
- 現金精算即時連携モジュール: Discord経由で外出先の現金支払いを即時登録します。

## 基本設計

ADAMS Liteは**モジュラーモノリス**構成を採用しています。
各機能はコード上でモジュールとして明確に分離しつつ、バックエンドサーバーは1つに統合することで、個人・家庭での運用・保守を容易にしています。
Discord BotはDocker Compose内の独立コンテナ（`discord-bot`）として動作し、ADAMS LiteのREST APIを経由してDBに書き込みます。自宅のUGREEN NAS上でDocker Composeにより他サービスと一元管理します。

### コンテナ構成

```
【Docker Compose】
nginx（リバースプロキシ）
  ├── frontend（React SPA）
  ├── backend（Express API）
  │     ├── budget モジュール（予算管理）
  │     ├── closing モジュール（決算管理）
  │     ├── events モジュール（行事計画）
  │     ├── funds モジュール（基金）
  │     └── reports モジュール（報告書）
  ├── db（MySQL 8.0）
  └── discord-bot（Discord Bot コンテナ）
        └── backend（POST /api/v1/actuals/discord）経由でDB書き込み
```

### ページ構成
- ホームページ（`/`）: 当月の予算実績サマリー・未締めアラート・進行中行事ウィジェットを表示します。
- 予算管理ページ: 予算の設定、管理、実績の追跡を行います。
- 決算管理ページ: 決算処理、報告書の生成を行います。
- 行事一覧ページ（`/events/plan`）: 行事の登録・管理・予算配分・仮押さえ確認を行います。
- 行事詳細ページ（`/events/plan/:id`）: 予算配分・支払計画・実績紐付け状況を確認します。
- ※旅費精算は独立ページなし。実績割り振り画面（`/budget/assign`）の行事フィルタで対応します。
- 基金管理ページ: 基金の積立・取崩・取引履歴・残高確認を行います。
- 設定ページ: 会計年度・ユーザー情報の管理を行います。

### API設計
- ダッシュボードAPI（`GET /fiscal-years/:fyId/dashboard`）: ホーム画面用。当月予算実績サマリー・進行中行事ウィジェットデータを返します。
- 予算管理API: 予算の作成、更新、削除、取得を行います。
- 決算管理API: 決算の作成、更新、削除、取得を行います。
- 行事計画API（`/events`）: 行事の登録・更新・ステータス管理・支払計画・予算配分・実績紐付けを行います。
- 基金API: 基金の作成、積立・取崩の記録、取引履歴参照（`GET /funds/:id/transactions`）、残高取得を行います。
- Discord連携API（`POST /api/v1/actuals/discord`）: Discord Botからの現金精算を受け付け、未割当実績として登録します。
