## 8. 現金精算即時連携モジュール

> **ステータス**: 要件定義中（Discord入力方式は **案②スレッド方式** に確定）

### 8.1 概要

外出先で発生した現金支払いを、その場でDiscordの個人サーバーに入力し、ADAMS Liteの未割当実績に即時反映する。現金支払いの記録漏れを防ぐことを目的とする。

### 8.2 操作シナリオ

1. 外出先で現金を支払う
2. スマートフォンでDiscordを開き、Bot に `/setevent イベント名` を送信（スレッドが作成される）
3. 作成されたスレッド内に `金額 メモ`（例: `500 コンビニ昼食`）を入力
4. Botが確認テロップを返信（取引日・金額・メモを表示）
5. 数分以内に修正がなければ自動確定 → ADAMS Lite APIに送信・未割当実績として登録
6. 帰宅後、通常の割り振り画面で科目に割り振り

### 8.3 Discord入力方式（**案②スレッド方式**に確定）

**概要**：`/setevent イベント名` でスレッドを作成し、そのスレッド内でテキスト形式（`金額 メモ`）で入力する。

```
【操作例】
ユーザー: /setevent 2月の買い物
Bot: 🧵 スレッド「2月の買い物」を作成しました。
     このスレッド内に「金額 メモ」の形式で入力してください。

--- スレッド内 ---
ユーザー: 500 コンビニ昼食
Bot: ✅ 確認テロップ → （8.6参照）
```

**検討した案との比較（参考）**

| 案 | 入力例 | 特徴 |
|---|---|---|
| A: テキスト（単体） | `500 コンビニ昼食` | 誤解析リスクあり |
| **② スレッド方式（確定）** | `/setevent 名前` → スレッド内で `500 昼食` | スレッドでイベント紐付け・テキスト入力の手軽さを両立 |
| C: コマンド+Modal | `/cash` → ポップアップ入力 | 2ステップで手間 |

### 8.4 登録データ仕様

| フィールド | 値 |
|---|---|
| date | Discord投稿日時（日本時間） |
| amount | 入力金額 |
| note | 入力メモ |
| direction | `歳出`（固定） |
| status | `unassigned`（未割当） |
| event_id | `/setevent` で指定したイベント（紐付けあり時） |

### 8.5 技術要素

| 項目 | 内容 |
|---|---|
| Bot実装 | Discord.js（最新版） |
| 連携先 | ADAMS Lite バックエンドAPI |
| APIエンドポイント（案） | `POST /api/v1/actuals/discord` |
| 認証 | Bot Token（Discord側）+ APIキー（ADAMS Lite側） |
| タイムアウト管理 | Bot側で投稿後タイマーを保持し、期限到達で自動確定APIを呼び出す |

### 8.6 確認テロップ仕様

入力を受け付けた直後、Botが以下の形式で確認メッセージをスレッドに返信する。

```
✅ 以下の内容で登録予定です。

取引日   ：令和8年02月27日（木）
金　額   ：500円
メ　モ   ：コンビニ昼食

⏱ 3分後に自動確定します。
修正する場合は「修正」、すぐに確定する場合は「決定」と返信してください。
```

**確定フロー**

| アクション | 動作 |
|---|---|
| タイムアウト（3分経過） | 自動確定 → `POST /api/v1/actuals/discord` 実行 |
| 「決定」と返信 | 即時確定（タイムアウト待ち不要） |
| 「修正」と返信 | Botが「何を修正しますか？金額・メモを再入力してください」と案内し、新しい入力を待機 |

> **タイムアウト時間**: 3分を想定（詳細設計フェーズで調整可）。外出先での入力を前提とするため、修正が不要な場合は操作ゼロで自動確定する設計とする。
