## 10. 非機能要件

| 区分 | 要件 |
|---|---|
| インフラ | 自宅サーバーで稼働（Docker Compose） |
| セキュリティ | JWT認証・HTTPS通信 |
| UI基準 | デジタル庁デザインシステム（DADS）準拠 |
| アイコン・イラスト | `materials/designsystem-assets/` 配下の素材を優先使用 |
| 基本色 | DADSカラーパレットを使用（CSS変数で管理） |
| レスポンシブ | PC・タブレット対応（スマートフォンは補助対応） |
| コード | 可能な限りTypeScriptで記述 |
