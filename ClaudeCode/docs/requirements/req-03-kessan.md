## 5. 決算管理モジュール

### 5.1 月次締め

| 項目 | 内容 |
|---|---|
| 実施タイミング | 毎月（例: 1月分 → 2月下旬〜3月上旬） |
| 処理内容 | 対象月の実績を確定し、以後の変更を不可とする |
| 未締めアラート | 前月以前の月次締めが未完了の場合、ホーム画面・ナビバーに警告を表示 |

### 5.2 年度締め（一般会計）

| 項目 | 内容 |
|---|---|
| 実施タイミング | 翌年度5月下旬〜6月上旬 |
| 処理内容 | 年度全体の実績を確定し、最終決算報告書を生成 |

### 5.3 予算執行状況の表示項目

Excelの「予算執行状況」シートに基づく。

| 項目 | 計算式 |
|---|---|
| 予算（当初） | 当初予算額 |
| 実績 | 割り振り済み実績の合計 |
| 予算実績差額 | 当初予算 − 実績 |
| 補正分 | 補正予算の累計増減額 |
| 補正後予算 | 当初予算 ＋ 補正分 |
| 補正後差額 | 補正後予算 − 実績 |
| **予算執行率** | 実績 ÷ 補正後予算 × 100（%） |

### 5.4 報告書出力

| 形式 | 内容 | 使用ライブラリ |
|---|---|---|
| 画面表示 | インタラクティブな予算実績対比表・グラフ | React（標準） |
| PDF出力 | 印刷対応の帳票形式（レイアウト詳細は別途決定） | **@react-pdf/renderer**（無料・MIT） |
| Excel出力 | .xlsx 形式（加工・分析用、レイアウト詳細は別途決定） | **ExcelJS**（無料・MIT） |

> **@react-pdf/renderer**: Reactコンポーネントとして帳票をデザインし、そのままPDF出力できる。フロントエンド側で完結。
> **ExcelJS**: バックエンド側で.xlsxを生成。罫線・セル結合・書式設定など帳票的な表現が得意。
